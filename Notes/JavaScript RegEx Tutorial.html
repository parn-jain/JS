What is a Regular Expression?

A Regular Expression (RegEx) is a pattern used to match character combinations in strings. In JavaScript, RegEx is an object that describes a search pattern.

How to Create Regular Expressions in JavaScript?

There are two ways:

  1. Using RegExp Constructor

js
const regex = new RegExp('pattern', 'flags');

  1. Using RegEx Literal Syntax

js
const regex = /pattern/flags;

Example:

js
const regex1 = new RegExp('hello');
const regex2 = /hello/;

RegEx Flags

Flags modify how the pattern is matched:

FlagMeaning

iCase-insensitive search
mMultiline mode
sDot (.) matches newline
uUnicode mode
ySticky search (matches from lastIndex)

Grouping and Capturing

  • Parentheses () group expressions and capture matching text.

Example:

js
const regex = /(abc)+/;
regex.test('abcabc');  // true

  • Non-capturing groups: (?:...) — groups without capturing.

Common Methods with RegEx in JavaScript

1. test()

Returns true or false if pattern exists in string.

js
const regex = /hello/;
console.log(regex.test('hello world')); // true
console.log(regex.test('goodbye'));     // false

2. exec()

Returns array with match info or null if no match.

js
const regex = /hello/;
const result = regex.exec('hello world');
console.log(result[0]); // "hello"

3. String.prototype.match()

Matches string against regex; returns array or null.

js
const str = "hello world";
const matches = str.match(/l+/g);
console.log(matches); // ["ll"]

4. String.prototype.replace()

Replaces matched substrings.

js
const str = "foo bar foo";
const newStr = str.replace(/foo/g, "baz");
console.log(newStr); // "baz bar baz"

5. String.prototype.search()

Returns index of match or -1.

js
const str = "hello world";
const index = str.search(/world/);
console.log(index); // 6

6. String.prototype.split()

Splits string by regex.

js
const str = "apple, banana; orange";
const parts = str.split(/[,;] /);
console.log(parts); // ["apple", "banana", "orange"]

Examples and Explanation

Match email

js
const emailRegex = /^[\w.-]+@[a-zA-Z\d.-]+\.[a-zA-Z]{2,6}$/;

  • ^ start, $ end

  • [\w.-]+ = word characters, dot or dash (user part)

  • @ literal @ symbol

  • [a-zA-Z\d.-]+ domain part

  • \. dot before domain suffix

  • [a-zA-Z]{2,6} domain suffix (e.g., com, org, net)

Validate phone number (US format)

js
const phoneRegex = /^\(?\d{3}\)?[-.\s]?\d{3}[-.\s]?\d{4}$/;

  • Optional parentheses for area code \(?\d{3}\)?

  • Optional separator (-, ., or space)

  • 3 digits + separator + 4 digits

Extract hashtags from a tweet

js
const hashtagRegex = /#\w+/g;
const str = "Loving #JavaScript and #regex!";
console.log(str.match(hashtagRegex)); // ["#JavaScript", "#regex"]

Escape Special Characters

If you want to match special characters literally, escape with \.

For example, to match a dot . literally:

js
const regex = /\./;

What are Groups in RegEx?

Parentheses () let you group parts of a pattern together. Grouping is useful to:

  • Apply quantifiers to the entire group.

  • Extract or capture parts of the matched string to use later.

Capturing Groups ( ... )

When you use normal parentheses like ( ... ), it captures the matched substring inside that group. This means:

  • The part of the string matched inside these parentheses is saved.

  • You can access it later from the result of your RegEx match.

Why is capturing useful?

Because when you run .exec() or .match(), you get an array where:

  • Index 0 = whole matched string.

  • Index 1 = first captured group.

  • Index 2 = second captured group, and so on.

Example: Capturing groups

js
const regex = /(\d{3})-(\d{4})/;
const result = regex.exec('My number is 123-4567');

console.log(result[0]); // "123-4567"  (whole match)
console.log(result[1]); // "123"       (first group)
console.log(result[2]); // "4567"      (second group)

Here:

  • (\d{3}) captures the first 3 digits.

  • (\d{4}) captures the next 4 digits.

Non-Capturing Groups (?: ... )

Sometimes you want to group part of the pattern (for applying quantifiers or logical grouping), but you don’t want to capture the matched substring. For that, use non-capturing groups: (?: ... ).

  • They group but do not save the match.

  • So, they won’t appear in the result array.

Why use non-capturing groups?

  • To improve performance if you don't need the capture.

  • To avoid cluttering the results with unnecessary groups.

  • To group for alternation or quantifiers without capturing.

Example: Non-capturing group

js
const regex = /(?:\d{3})-(\d{4})/;
const result = regex.exec('My number is 123-4567');

console.log(result[0]); // "123-4567"  (whole match)
console.log(result[1]); // "4567"      (only one captured group)
console.log(result[2]); // undefined   (no second capture)

Here:

  • (?:\d{3}) matches 3 digits but does NOT capture them.

  • (\d{4}) still captures the last 4 digits.

Key differences summarized

AspectCapturing Group ( ... )Non-Capturing Group (?: ... )
Captures matched textYesNo
Available in resultYes, as numbered groupsNo
Used for groupingYesYes
Use when you wantTo extract parts of stringTo group but ignore the matched text

One more example comparing both:

js
const capturing = /(foo|bar)/;
const nonCapturing = /(?:foo|bar)/;

console.log("abc foo def".match(capturing));
// Output: ["foo", "foo"]

console.log("abc foo def".match(nonCapturing));
// Output: ["foo"]

  • The capturing group returns both the full match and the captured group.

  • The non-capturing group returns just the full match.

TL;DR:

  • Use ( ... ) if you want to save part of the match to use later.

  • Use (?: ... ) if you just want to group for logic/quantifiers but don't need to save the match.
